# 会話ログの記録と再利用マニュアル
**Claudeとのやりとりを残して、次回から活用しましょう**

---

## STEP 1：会話を記録する

Claudeとの作業が一区切りついたら、以下のように伝えてください。

### そのまま使える指示フレーズ

**全文をそのまま残したいとき：**
```
ここまでの会話内容を一字一句変えずに conversation-2026-04-02.md というファイル名で保存して
```

**要点だけまとめてほしいとき：**
```
今日のやりとりを以下の形式でまとめて、conversation-2026-04-02.md として保存して

- 日付
- 業務テーマ
- やりとりの要点（箇条書き）
- うまくいったプロンプト
- 気づき・学び
```

**うまくいったプロンプトだけ残したいとき：**
```
今日のやりとりの中で、うまくいったプロンプトだけ抜き出してファイルに保存して
```

> **ポイント：** 日付部分は、その日の日付に変えてください。ファイル名は日本語を避けて、英数字にしましょう。

---

## STEP 2：保存されたことを確認する

1. GitHub のリポジトリページをブラウザで開く
2. `participants` → 自分のフォルダ を開く
3. 新しいファイルが追加されていれば OK

※ 保存先やファイル名が違っていたら、Claudeに「participants/（自分のフォルダ）/ に保存して」と伝え直せば大丈夫です。

---

## STEP 3：次回、過去の記録を活用する

新しくClaudeを開くと、前回の会話内容は残っていません。
GitHubに保存した記録をClaudeに読み込ませることで、前回の続きから作業できます。

### 3-1. リポジトリをClaudeに教える

新しい会話を始めたら、最初にリポジトリのURLを伝えます。

```
このリポジトリで作業したい
https://github.com/gate-wing/demo-repository
```

> Claudeがリポジトリの内容を把握してくれます。初回はClaudeから名前の確認があるので、答えてください。

### 3-2. 前回の記録を読み込む

自分のフォルダにある過去の記録を読んでもらいます。

```
participants/（自分のフォルダ）/ にあるファイルを確認して、前回の内容を把握して
```

特定のファイルを指定してもOKです。
```
participants/（自分のフォルダ）/conversation-2026-04-02.md を読んで
```

### 3-3. 前回の続きから作業する

記録を読み込んだ後、やりたいことを伝えましょう。

**続きをやりたいとき：**
```
前回の内容を踏まえて、同じ業務の続きを進めたい
```

**前回のプロンプトを改善したいとき：**
```
前回うまくいったプロンプトをベースに、もっと良くできる点があれば改善案を出して
```

**他の人の記録を参考にしたいとき：**
```
participants/ の中で、他の人がどんなプロンプトを作っているか見せて
```

### まとめ：次回の会話の始め方

| 順番 | やること | 伝えるフレーズ |
|:---:|---------|---------------|
| 1 | リポジトリを指定する | `https://github.com/gate-wing/demo-repository で作業したい` |
| 2 | 前回の記録を読み込む | `participants/（自分のフォルダ）/ のファイルを確認して` |
| 3 | やりたいことを伝える | `前回の続きをやりたい` など |

---

## 記録ファイルの命名ルール

| 種類 | ファイル名の例 |
|------|---------------|
| 日付ごとの記録 | `conversation-2026-04-02.md` |
| テーマ別の記録 | `conversation-weekly-report.md` |
| プロンプト集 | `prompts-email-templates.md` |

---

## よくある質問

**Q. 毎回記録しないとダメ？**
→ 必須ではありません。「これは残しておきたい」と思ったときだけでOKです。

**Q. 失敗したやりとりも残す意味はある？**
→ あります。「この指示だとうまくいかなかった」という記録は、次回の改善に役立ちます。

**Q. 他の人の記録を見てもいい？**
→ はい。`participants` フォルダ内の他の方のファイルも自由に見られます。他の人のプロンプトを参考にするのはとても効果的です。
