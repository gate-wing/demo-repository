# GitHub ブラウザ操作チートシート
**印刷してお手元に置いておいてください**

---

## ファイルを新しく保存する手順

### STEP 1：自分のフォルダを開く

1. ブラウザでGitHubのリポジトリページを開く
2. `participants` フォルダをクリック
3. 自分のフォルダ名をクリックして開く

---

### STEP 2：新しいファイルを作る

1. 右上の **[Add file]** ボタンをクリック
2. **[Create new file]** を選択

---

### STEP 3：ファイル名と内容を入力する

1. 上部の入力欄にファイル名を入力する
   - 例：`daily-report-prompt.md`
   - **日本語のファイル名は避ける**（トラブルの原因になることがあります）
   - 末尾は必ず `.md` をつける

2. 下の大きな入力欄に、Claudeで完成させたプロンプトや手順を貼り付ける

---

### STEP 4：保存する（コミット）

1. 画面を下にスクロールする
2. **[Commit changes]** ボタンをクリック
3. 確認のダイアログが出たら、もう一度 **[Commit changes]** をクリック

✅ これで保存完了です！

---

## 既存のファイルを編集する手順

1. 編集したいファイルを開く
2. 右上の **鉛筆アイコン（✏️）** をクリック
3. 内容を編集する
4. 下にスクロールして **[Commit changes]** をクリック

---

## よくあるトラブルと対処法

| 状況 | 対処法 |
|------|--------|
| ファイルが見つからない | `participants` フォルダの中に自分のフォルダがあるか確認する |
| 保存できない | ファイル名が空になっていないか確認する |
| 文字が崩れた | ファイル名に日本語や記号が含まれていないか確認する |
| ページが真っ白になった | ブラウザを更新する（F5 キー） |

---

## 覚えておくと便利な言葉

| 言葉 | 意味 |
|------|------|
| リポジトリ | ファイルをまとめて保管する「共有フォルダ」のこと |
| コミット | ファイルを保存・確定する操作のこと |
| ブランチ | 変更作業用の「下書きスペース」のこと（今回は使いません） |
| `.md` ファイル | 見出しや箇条書きが使えるメモ形式のテキストファイル |
